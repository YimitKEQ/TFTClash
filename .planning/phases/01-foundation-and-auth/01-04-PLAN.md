  <name>Task 1: Create Vercel deployment config and production env checklist</name>
  <files>vercel.json, next.config.js, .env.production.example</files>
  <action>
    **vercel.json** — Minimal Vercel config for Next.js App Router (no SPA rewrites needed — App Router handles routing natively):
    ```json
    {
      "framework": "nextjs",
      "buildCommand": "npm run build",
      "outputDirectory": ".next"
    }
    ```
    Keep it minimal. Do NOT add SPA redirect rules (those were for the old Vite prototype's vercel.json — App Router does not need them).

    **next.config.js** — Ensure it includes:
    - `images.domains` or `images.remotePatterns` allowing `cdn.discordapp.com` (Discord avatar URLs from OAuth)
    - `env.NEXT_PUBLIC_SITE_URL` referencing the environment variable

    Example:
    ```js
    /** @type {import('next').NextConfig} */
    const nextConfig = {
      images: {
        remotePatterns: [
          { protocol: 'https', hostname: 'cdn.discordapp.com' },
        ],
      },
    };
    module.exports = nextConfig;
    ```

    **.env.production.example** — Human-readable checklist to set in Vercel dashboard:
    ```
    # Copy these to Vercel Dashboard -> Project -> Settings -> Environment Variables
    # Set environment to "Production" (and optionally "Preview")

    NEXT_PUBLIC_SUPABASE_URL=https://your-project-ref.supabase.co
    NEXT_PUBLIC_SUPABASE_ANON_KEY=your-anon-key
    SUPABASE_SERVICE_ROLE_KEY=your-service-role-key

    # Your production domain (used for OAuth redirect construction)
    NEXT_PUBLIC_SITE_URL=https://tftclash.gg

    # After setting env vars, also update in Supabase Dashboard:
    # Authentication -> URL Configuration -> Site URL: https://tftclash.gg
    # Authentication -> URL Configuration -> Redirect URLs: https://tftclash.gg/auth/callback
    ```
